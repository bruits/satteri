Nodes retained past their visitor pass now read as the tree looked during that pass instead of unconditionally throwing. Any in-pass resolve of node content (reading a child node's field, or `ctx.parent()`/`ctx.indexOf()`) pins the pass's arena snapshot, and retained nodes resolve against it even after later plugins mutate the tree; with no pinned snapshot, reads still work as long as nothing has mutated or freed the arena yet. The retention error only remains for the genuinely unrecoverable case (an untouched node first read after the tree changed) and now explains how to avoid it. Materialized node objects are also shared rather than copied: the same underlying node reached through any path, including a later pass over an unchanged tree, is the same JavaScript object, so treat nodes as read-only data and mutate through the context methods. Retained nodes keep their pass snapshot alive until collected, so copying the data you need (`structuredClone(node)`) is still the cheaper way to keep results around.
